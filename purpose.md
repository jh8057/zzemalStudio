# 개발적 요구

1. 기존에 three.js를 이용해서 메인 배경을 만들었는데, 개편하고 싶어.

- three.js를 이용한 더 좋은 방법을 찾고 싶어.
- 오브제를 활용해서 통통 튀는 느낌.

2. 서버리스로 aws 를 적극 활용중이야. 이건 계속 그랬으면 좋겠어.

3. cypress 같은 e2e 테스트 코드 제거해. 단위 테스트만 있으면 돼.

4. 스튜디오 사이트에서 자주 쓰는 기능들이 있다면 추가됐으면 좋겠어.

5. 어드민 페이지나 혹은 별도 내가 개인적으로 쓸 수 있는 환경도 있으면 좋겠네.

6. 이왕 바꾸는거 react 코드로 바꿀까해. 지금 s3s 에 빌드해서 정적페이지로 쓰고 있는데, 기존 방식 유지 가능한지 체크해주고.

# 디자인 요구

없어. 맘대로해

스튜디오 달리 웹사이트 개편 계획

Context

스튜디오 달리(증명사진/여권사진/프로필 촬영 전문, 서울 강서구 염창동)의 웹사이트를 Vue 3에서 Next.js로 전면 개편한다.
현재 사이트는 오래된 느낌이 있고, 최신 스튜디오 사이트의 깔끔하고 전문적인 분위기를 목표로 한다.
콘텐츠는 Instagram/네이버 연동 중심, 예약은 네이버 예약이 우선이며, AWS 서버리스 인프라를 유지한다.

페이지 설계

홈 (/)

1.  Hero - 전체 화면. Three.js Canvas 배경 (카메라 렌즈 + 통통 튀는 도형). 스튜디오 이름 + 태그라인
2.  서비스 소개 - 3개 카드: 증명사진, 여권사진, 프로필 촬영 (아이콘 + 설명 + 가격 범위)
3.  Instagram 피드 - Instagram 프로필 임베드 위젯 (최근 포스트 자동 표시, 포트폴리오 미리보기 겸용)
4.  네이버 리뷰 - 네이버 플레이스 평점 표시 + "리뷰 보기" 링크 버튼
5.  CTA - "예약하기" 버튼 (네이버 예약으로 이동)

포트폴리오 (/portfolio)

- Instagram 프로필 임베드 위젯을 메인 갤러리로 사용 (API 연동 없음, 수동 관리 없음)
- Instagram에 새 사진 올리면 사이트에 자동 반영
- "Instagram에서 더 보기" 링크
- 별도 사진 업로드/관리 불필요

예약안내 (/reserve)

- 가격표 카드 (평일 25,000원 / 주말 30,000원)
- 네이버 예약 버튼 (메인 CTA) - 네이버 예약 페이지로 이동
- 직접 예약 폼 (보조) - 기존 Lambda 활용
- 이용안내 (입금정보, 취소정책, 주의사항)

오시는길 (/location)

- 카카오/네이버 지도 임베드
- 주소 복사 기능
- 네이버 지도/카카오 지도 딥링크

어드민 (/admin) - Phase 3

- 단순 비밀번호 로그인 (환경변수에 비밀번호 해시 저장, OAuth/소셜 로그인 없음)
- 예약 목록 조회/관리 (DynamoDB)
- 예약 상태 변경 (대기/확인/취소)
- 별도 승인 절차나 외부 서비스 연동 없이 자체 인증만 사용

Three.js (홈 페이지만)

React Three Fiber 사용. 'use client' 지시문 필수.

컨셉: 카메라 렌즈 + 통통 튀는 도형

- 중앙에 스타일화된 카메라 렌즈 (torus geometry + 유리 재질) 천천히 회전
- 주변에 작은 기하학 도형(icosahedron, sphere)이 Float 컴포넌트로 부드럽게 부유
- MeshDistortMaterial로 유기적인 느낌
- 마우스 패럴랙스 효과
- 모노크롬 + 포인트 컬러 (따뜻한 앰버 또는 쿨 블루)

성능 고려:

- SSG 빌드 시 빈 div로 렌더 → 브라우저에서 초기화
- 히어로 CSS 그라디언트 배경으로 로딩 중 빈 화면 방지
- 모바일: 저사양 기기 감지 시 Three.js 비활성화 또는 간소화

외부 콘텐츠 연동 (API 연동 없이, 임베드 방식)

Instagram/네이버 계정에 장기 접근이 어려우므로, API 연동 없이 임베드 위젯과 링크로 처리한다.

Instagram (포트폴리오 + 홈 피드)

- Instagram 임베드 위젯 사용 (API 키, 토큰, Meta Developer App 모두 불필요)
- instagram.com/studio.darly/embed/ iframe으로 프로필 피드 임베드
- 홈 페이지: Instagram 프로필 위젯 (최근 포스트가 자동으로 표시됨)
- 포트폴리오 페이지: Instagram 프로필 임베드를 메인 갤러리로 사용
- "Instagram에서 더 보기" 링크로 전체 프로필 연결
- 장점: 새 포스트 올리면 사이트에 자동 반영, 빌드/배포 불필요

네이버 리뷰

- 네이버 플레이스 리뷰 링크 버튼 제공 ("네이버 리뷰 보기")
- 네이버 플레이스 평점/리뷰 수를 표시하려면 Lambda 스크래핑 가능 (공개 페이지라 로그인 불필요)
  - Lambda fetchNaverRating: 공개 네이버 Place 페이지에서 평점/리뷰 수만 추출
  - 빌드 시 public/data/naver-rating.json 생성
  - 스크래핑 실패 시 fallback으로 정적 데이터 표시
- 또는 단순하게 네이버 플레이스 링크 버튼만 제공

콘텐츠 갱신

- Instagram: 임베드 위젯이라 실시간 반영 (빌드 불필요)
- 네이버 평점: 스크래핑 사용 시 주간 스케줄 빌드, 링크만 사용 시 빌드 불필요

예약 시스템

- 주 채널: 네이버 예약 (외부 링크)
- 보조 채널: 직접 예약 폼 → 기존 Lambda 업데이트
  - GET → POST 변경
  - DynamoDB 저장 (어드민 조회용)
  - SES 이메일 알림
- FullCalendar 제거 (네이버 예약이 source of truth)

Lambda API 구조

POST /reservations → 예약 생성 (DynamoDB 저장 + SES 알림)
POST /admin/login → 비밀번호 검증, JWT 발급
GET /admin/reservations → 예약 목록 조회 (JWT 필요)
PATCH /admin/reservations/:id → 예약 상태 변경 (JWT 필요)

---

배포

S3 + CloudFront

- S3: 정적 웹사이트 호스팅, CloudFront OAC로 접근 제어
- CloudFront: SSL(ACM), Gzip/Brotli 압축, 캐시 전략
- Route53: studiodarly.com → CloudFront alias

GitHub Actions

push to main → npm ci → build → s3 sync → CloudFront invalidation
Instagram은 임베드라 빌드 시 데이터 페치 불필요. 콘텐츠가 실시간 반영됨.

검증 방법

1.  npm run dev → 로컬에서 모든 페이지 확인
2.  npm run build → 정적 빌드 성공 확인 (out/ 폴더 생성)
3.  npx serve out → 정적 빌드 결과물 로컬 테스트 (라우팅, 404 등)
4.  npm test → Vitest 단위 테스트 통과
5.  Lighthouse 감사 → Performance 90+, Accessibility 90+
6.  모바일 실기기 테스트 → Three.js 성능, 터치 내비게이션
7.  S3 배포 후 CloudFront URL로 접근 테스트
